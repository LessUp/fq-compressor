# Academy

The academy lane is for execution. Use it when the question is how to install, run, verify, or contribute without
losing the architectural and evidence context around the tool.

## Curriculum

- [Installation](./installation): choose between the published Linux binary and a local source build.
- [Getting started](./getting-started): run a verified smoke-scale round trip after installation.
- [CLI workflows](./cli-workflows): the compact command surface for `compress`, `decompress`, `info`, and `verify`.
- [Contributing](./contributing): closeout-mode workflow, quality gates, and documentation expectations.

## How to use this section

Start here when you already accept the project framing and need an execution path. Keep [Architecture](/en/architecture/)
nearby when an option needs deeper implementation context, and keep [Research](/en/research/) nearby when you need
external references or provenance.
