# Evolution notes

fq-compressor is in **closeout mode**. That changes both engineering priorities and documentation priorities.

## What closeout mode means here

- Finishing, simplifying, and stabilizing beats adding new product surface area.
- Public claims should get sharper, not bigger.
- Evidence quality, architecture clarity, and contributor onboarding matter more than feature expansion.

## What that changes in the docs

- The site behaves like a whitepaper and research portal, not a generic README mirror.
- Academy and research sections replace weaker catch-all labels because readers arrive with different intents.
- Evidence language is kept narrower than aspiration language.

## What remains intentionally deferred

- Broad best-in-class claims that exceed the current measured proof surface.
- Expanding the site into a heavy frontend application with bespoke JS behavior.
- Treating archived material as if it were live public documentation.
