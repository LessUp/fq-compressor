# 核心算法

fq-compressor 使用针对不同数据流优化的**混合压缩策略**。

## 概览

| 数据流 | 算法 | 描述 |
|--------|-----------|-------------|
| 序列 | **ABC** | 基于组装的压缩 |
| 质量值 | **SCM** | 统计上下文混合 |
| ID | Delta+Token | 分词和增量编码 |

---

## ABC：基于组装的压缩

### 原理

将读段视为基因组片段而非独立字符串。通过以下方式利用生物学冗余：
1. 分组相似读段
2. 生成共识序列
3. 编码差异（增量）

### 流水线

```
输入读段
  ↓
1. Minimizer 索引 (k=21, w=11)
  ↓
2. 分桶（按共享 minimizer）
  ↓
3. 重排序（TSP 近似）
  ↓
4. 共识生成
  ↓
5. 增量编码
  ↓
压缩输出
```

### 1. Minimizer 索引

**Minimizer** 是连续 w 个 k-mer 窗口中字典序最小的 k-mer。

```cpp
for each window of w k-mers:
    minimizer = min(window)
    index[minimizer].add(read_id)
```

**参数：**
- `k = 21`：k-mer 大小
- `w = 11`：窗口大小

**优势：**
- 相比存储所有 k-mer 减少内存
- 保持相似读段的敏感性
- 生物学相关性（minimizer 聚类）

### 2. 分桶

共享 minimizer 的读段被分组。目标桶大小：~10,000 读段。

**原因：**
- 相同基因组区域 → 共享 minimizer
- 局部共识比全局更准确
- 支持并行按桶处理

### 3. 重排序（TSP）

贪心近似最小化连续读段间编辑距离：

```python
def reorder(bucket):
    ordered = [bucket.pop_random()]
    while bucket:
        last = ordered[-1]
        next_read = min(bucket, key=lambda r: edit_distance(last, r))
        ordered.append(next_read)
    return ordered
```

**复杂度：** 每桶 O(n²)

### 4. 共识生成

从比对读段进行位置多数投票：

```python
def generate_consensus(ordered_reads):
    alignment = multi_align(ordered_reads)
    consensus = []
    for position in alignment.columns:
        counts = Counter(position.bases)
        consensus.append(counts.most_common(1)[0][0])
    return ''.join(consensus)
```

### 5. 增量编码

将读段编码为相对共识的编辑：

```cpp
enum class EditType { Match, Subst, Insert, Delete };

struct Edit {
    EditType type;
    uint32_t position;
    char base;  // 用于 Subst/Insert
};
```

**示例：**
```
共识: ACGTACGTACGTACGT
读段: ACGTAAGTACGTACGT

编辑: [6]: Subst C → A
```

**压缩比：** 基因组序列通常 3-5×

## SCM：统计上下文混合

### 原理

高质量值使用高阶上下文建模和算术编码。

### 上下文建模

基于以下预测下一个质量值：
- 前 2 个质量值
- 当前序列碱基
- 读段内位置

```cpp
struct Context {
    uint8_t qv_minus_1;
    uint8_t qv_minus_2;
    char current_base;
    uint8_t position_bucket;
};
```

### 模型混合

组合多个上下文阶的预测：

```python
# Order-0: P(Q_i)
# Order-1: P(Q_i | Q_{i-1})
# Order-2: P(Q_i | Q_{i-1}, Q_{i-2})
# Order-s: P(Q_i | sequence_context)

final = sum(weight[i] * model[i].predict(context[i])
            for i in range(num_models))
```

### 算术编码

将预测转换为压缩比特：

```
质量值: [33, 34, 35, 36, 37]
计数:   [100, 200, 400, 200, 100]

P(35) = 400/1000 = 0.4
Bits(35) = -log2(0.4) ≈ 1.32 bits
```

### 质量分桶（可选）

通过分桶进行有损压缩：

| 方法 | 桶数 | 比特/QV | 影响 |
|--------|------|---------|--------|
| 无损 | 94 | ~5 | 无 |
| Illumina 8 | 8 | 3 | 最小 |
| 4 桶 | 4 | 2 | 中等 |

## ID 压缩

### 分词

按模式转换分割读段 ID：

```
@E150035817L1C001R0010000000/1
→ ['E', '150035817', 'L', '1', 'C', '001', 'R', '001', '0000000', '/', '1']
```

### 增量编码

用于单调数字分词：

```
原始: [001, 002, 003, 004, 005]
增量: [001, 001, 001, 001, 001]
```

### 字典

为频繁模式构建常用分词字典。

## 性能总结

| 算法 | 速度 | 压缩比 | 内存 | 并行 |
|-----------|-------|-------|--------|----------|
| ABC | ~10 MB/s | 3-5× | 中等 | 桶级 |
| SCM | ~15 MB/s | 2-3× | 低 | 是 |
| ID | ~50 MB/s | 5-10× | 低 | 是 |

**综合：** ~11 MB/s 压缩，~4× 典型压缩比
