# FQC 归档格式

**FQC** (FASTQ Compressed) 格式是针对基因组 FASTQ 数据优化的领域专用归档格式。

## 特性

- **分块存储**：独立的压缩块
- **列式流**：分离的 ID/序列/质量值压缩
- **随机访问**：O(1) 提取任意读段范围
- **完整性验证**：CRC32C 校验和
- **重排序支持**：原始顺序恢复

## 文件结构

```
┌─────────────────────────────────────────────┐
│ FQC 归档                                    │
├─────────────────────────────────────────────┤
│ 固定头部 (256 字节)                         │
│   - Magic: "FQCC"                           │
│   - Version: 0.1.0                          │
│   - 格式标志                                │
│   - 块数量                                  │
│   - 总读段/碱基数                           │
├─────────────────────────────────────────────┤
│ 压缩参数                                    │
├─────────────────────────────────────────────┤
│ Block 0                                     │
│   ┌─ 块头部 (64 字节)                       │
│   ├─ ID 流 (zstd)                           │
│   ├─ Seq 流 (ABC+zstd)                      │
│   └─ Qual 流 (SCM+zstd)                     │
├─────────────────────────────────────────────┤
│ Block 1 ...                                 │
├─────────────────────────────────────────────┤
│ Block N-1                                   │
├─────────────────────────────────────────────┤
│ 页脚                                        │
│   - 块索引表                                │
│   - 重排序映射（可选）                      │
│   - 页脚校验和                              │
└─────────────────────────────────────────────┘
```

## 固定头部

```c
struct FQCHeader {
    char magic[4] = {'F', 'Q', 'C', 'C'};
    uint16_t version_major = 0;
    uint16_t version_minor = 1;
    uint16_t version_patch = 0;
    uint16_t format_flags;
    uint32_t block_count;
    uint64_t total_reads;
    uint64_t total_bases;
    uint8_t reserved[224];
};
```

### 格式标志

| 位 | 名称 | 描述 |
|-----|------|-------------|
| 0 | PAIRED_END | 双端数据 |
| 1 | REORDERED | 读段已重排序 |
| 2 | ORIG_ORDER_PRESERVED | 可恢复原始顺序 |
| 3 | HAS_METADATA | 包含元数据块 |

## 块格式

### 块头部

```c
struct BlockHeader {
    uint32_t block_id;
    uint32_t flags;
    uint64_t original_size;
    uint64_t compressed_size;
    uint32_t read_count;
    uint64_t base_count;
    uint32_t crc32c_data;
    uint32_t crc32c_uncompressed;
};
```

### 块数据

每个块包含三个压缩流：

| 流 | 编码 | 内容 |
|--------|----------|---------|
| ID | Token + Delta + Zstd | 读段标识符 |
| 序列 | ABC + Zstd | 相对共识的增量 |
| 质量值 | SCM + Zstd | 上下文混合分数 |

## 页脚

### 块索引表

```c
struct BlockIndexEntry {
    uint64_t offset;          // 文件偏移
    uint64_t size;            // 块总大小
    uint32_t read_count;      // 读段数量
    uint64_t first_read_id;   // 全局读段 ID
};
```

### 重排序映射

将压缩顺序映射到原始顺序：

```c
struct ReorderMap {
    uint32_t version = 1;
    uint64_t read_count;
    uint32_t compression_type;
    uint8_t compressed_mapping[];  // 增量编码变长整数
};
```

## 实现说明

### 块大小

- **默认**：每块 ~10MB 未压缩
- **更小块**：更好的随机访问，更多开销
- **更大块**：更好的压缩，更差的访问

### 压缩级别

| 模式 | Zstd 级别 | 用例 |
|------|------------|----------|
| fast | 1 | 速度优先 |
| balanced | 3 | 默认 |
| best | 19 | 存储优先 |

### 内存映射

块头部对齐到 4096 字节（页大小）以支持高效内存映射。
