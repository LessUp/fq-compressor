# 学院

学院路径面向执行。只要你的问题是如何安装、运行、校验或贡献，并且又不想脱离架构和证据语境，就从这里开始。

## 课程目录

- [安装](./installation)：在 Linux 发布二进制和本地源码构建之间做选择。
- [快速开始](./getting-started)：安装完成后，做一次可验证的烟雾级往返。
- [CLI 工作流](./cli-workflows)：`compress`、`decompress`、`info`、`verify` 的紧凑命令面。
- [贡献](./contributing)：closeout 模式下的工作流、质量门槛与文档要求。

## 如何使用本节

当你已经接受项目定位、只需要一条可执行路径时，就从这里开始。如果某个选项需要系统上下文，请同时参考 [架构](/zh/architecture/)；如果需要外部文献或出处，请同时打开 [研究](/zh/research/)。
