# 演进思考

`fq-compressor` 目前处于 **closeout mode**。这会同时改变工程优先级和文档优先级。

## 这在这里意味着什么

- 优先完成、简化和稳定，而不是继续增加产品表面积。
- 公开主张应该更精确，而不是更夸张。
- 证据质量、架构清晰度和贡献者导读，比新功能扩张更重要。

## 这如何改变文档

- 站点更像白皮书与研究入口，而不是 README 镜像。
- 用“学院”和“研究”替代更松散的聚合标签，因为不同读者带着不同问题而来。
- 证据语言必须比愿景语言更收敛。

## 哪些内容被刻意延后

- 超出当前实测证明面的 best-in-class 公开结论。
- 把站点扩张成重前端应用，增加大量定制 JS 行为。
- 把归档材料当成现行公开文档来使用。
