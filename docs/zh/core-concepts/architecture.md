# 架构概述

> 了解 fq-compressor 的高层设计和组件

## 设计目标

fq-compressor 的设计围绕以下关键目标：

1. **高压缩比**: 接近基因数据的理论熵极限
2. **并行性能**: 最大化现代多核处理器的利用率
3. **随机访问**: 实现任何 reads 子集的 O(1) 访问
4. **工业级质量**: 生产就绪的错误处理、日志记录和内存管理
5. **可维护性**: 清晰、现代的 C++23 代码，关注点分离明确

## 组件层

### 1. CLI 层

CLI 层处理命令行参数解析和命令路由。

**关键组件:**
- `main.cpp`: 入口点，全局选项处理
- CLI11 库: 类型安全的命令行解析

**职责:**
- 解析命令行参数
- 验证输入
- 分发到适当的命令处理器
- 处理全局选项（线程、详细程度等）

### 2. 命令层

将各个 CLI 命令实现为独立的类。

**命令:**
- `CompressCommand`: 单遍和两阶段压缩
- `DecompressCommand`: 完整和部分解压
- `InfoCommand`: 归档元数据检查
- `VerifyCommand`: 完整性验证
- `SplitPeCommand`: 双端测序分离

**设计模式**: 命令模式，统一错误处理

### 3. 流水线层

使用 Intel oneTBB 进行并行处理。

**流水线阶段:**
1. **生产者节点**: 从输入读取 FASTQ 记录
2. **压缩器节点**: 并行压缩块
3. **写入器节点**: 将压缩块写入输出

**主要特性:**
- 基于令牌的流控制（限制内存使用）
- 工作窃取调度器实现负载均衡
- 从工作线程传播异常

### 4. 格式层

FQC 归档格式实现。

**关键组件:**
- `FQCWriter`: 将压缩块序列化为 FQC 格式
- `FQCReader`: 反序列化 FQC 归档
- `ReorderMap`: 管理 reads 重排序用于压缩/恢复
- `BlockIndex`: 实现 O(1) 随机访问

**FQC 格式特性:**
- 基于块的存储（通常 10MB 未压缩）
- 列式流（分离的 ID、序列、质量流）
- 用于完整性验证的校验和
- 用于原始顺序恢复的重排序映射

### 5. 算法层

核心压缩算法。

#### 序列压缩 (ABC)
- `GlobalAnalyzer`: 基于 minimizer 的分桶、reads 重排序
- `SequenceCompressor`: 相对共识的增量编码

#### 质量压缩 (SCM)
- `QualityCompressor`: 统计上下文混合
- `ArithmeticCoder`: 高精度熵编码

#### ID 压缩
- `IDCompressor`: 标记化和增量编码

### 6. I/O 层

具有格式抽象的高性能 I/O。

**特性:**
- 输入文件的透明解压（.gz, .bz2, .xz）
- 基于块的压缩输出（zstd, zlib）
- 带预取的异步 I/O
- 缓冲写入

## 数据流

### 压缩流程

```
FASTQ 输入
    │
    ▼
┌─────────────┐
│ FastqParser │ (透明解压)
└──────┬──────┘
       │
       ▼
┌───────────────┐     ┌───────────────────┐
│GlobalAnalyzer │────▶│ Reorder Map       │
│ (分桶)        │     │ (保存到归档)      │
└───────┬───────┘     └───────────────────┘
        │
        ▼
┌───────────────┐
│ BlockBuilder  │ (分组为块)
└───────┬───────┘
        │
        ▼
┌───────────────────────────────────────┐
│ BlockCompressor                       │
│   ├── IDCompressor (delta+token)     │
│   ├── SequenceCompressor (ABC)       │
│   └── QualityCompressor (SCM)        │
└───────────────┬───────────────────────┘
                │
                ▼
┌─────────────┐
│ FQCWriter   │ (带索引序列化)
└──────┬──────┘
       │
       ▼
    .fqc 文件
```

        ▼

## 内存管理

### 内存预算系统

fq-compressor 具有自动内存预算系统：

1. **检测**: 自动检测可用系统内存
2. **分配**: 在子系统间分配预算
3. **节流**: 在流水线中使用基于令牌的反压
4. **限制**: 尊重用户定义的 `--memory-limit`

### 各组件内存使用

| 组件 | 典型使用 | 可配置 |
|------|----------|--------|
| 流水线令牌 | ~10% 预算 | 是（间接） |
| 块缓冲 | ~30% 预算 | 是（块大小） |
| 排序/重排序 | ~40% 预算 | 否 |
| 压缩上下文 | ~20% 预算 | 否 |

## 线程模型

### oneTBB 集成

- **流图**: 生产者 → 压缩器 → 写入器流水线
- **并行算法**: 块处理的 parallel_for
- **并发容器**: 工作分发的 concurrent_queue

### 线程安全保证

- **解析**: 单线程（带预读）
- **压缩**: 每块线程安全
- **写入**: 线程安全且保持顺序

## 错误处理策略

### 错误类别

| 类别 | 处理 | 示例 |
|------|------|------|
| 解析错误 | 致命，带位置 | 格式错误的 FASTQ |
| I/O 错误 | 致命，带路径 | 权限被拒绝 |
| 内存错误 | 致命，带统计 | 内存不足 |
| 格式错误 | 致命，带上下文 | 损坏的归档 |

### 错误报告

- **结构化**: `Result<T>` / `VoidResult` 类型
- **上下文**: 源代码位置、文件位置
- **日志**: 使用 Quill 的结构化日志
- **CLI**: 用户友好的错误消息

## 扩展点

3. 在 `main.cpp` 中注册

1. 实现 `CompressorInterface`
2. 更新 `BlockCompressor` 分发
3. 添加版本处理以保持向后兼容
