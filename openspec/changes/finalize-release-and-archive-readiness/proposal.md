## Why

The repository owner wants to bring `fq-compressor` to a polished low-maintenance state without
publicly repositioning it as abandoned. That requires an explicit internal closeout checklist for
release quality, repository clarity, and archive readiness.

## What Changes

- Define the archive-ready checklist for the repository.
- Align GitHub metadata and release surfaces with the final project story.
- Ensure final closeout leaves no unmanaged active changes or lingering local cleanup debt behind.

## Capabilities

### New Capabilities

- `release-readiness`: defines the repository state required before entering low-maintenance mode.

### Modified Capabilities

- `compatibility`: clarify what final validation and metadata checks are required before closeout.

## Impact

- OpenSpec closeout changes and tasks
- GitHub repository description, homepage, and topics
- Release and archival process documentation
- Internal closeout checklist and review flow
