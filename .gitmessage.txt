# <type>(<scope>): <subject>
#
# type: 
#   feat     : 新功能 (feature)
#   fix      : 修补 bug
#   docs     : 文档 (documentation)
#   style    : 格式 (不影响代码运行的变动)
#   refactor : 重构 (既不是新增功能，也不是修改 bug 的代码变动)
#   perf     : 提高性能的优化
#   test     : 增加测试
#   build    : 影响构建系统或外部依赖的更改 (如: gulp, npm)
#   ci       : 更改 CI 配置和脚本 (如: Travis, Circle)
#   chore    : 非 src 或测试文件的修改 (如: gitignore)
#   revert   : 回滚到上一个版本
#
# scope: 可选，模块/目录/组件名（如 cli, io, compress, abc, scm）
# subject: 简短描述（不以句号结尾）
#
# ---------------------
# Body（可选）：
# - 说明动机/实现要点/风险
#
# Footer（可选）：
# - Breaking Change: ...
# - Closes #123
#
# ---------------------
# 如何强制推行？
# 1. 设置本地模板: git config --local commit.template .gitmessage.txt
# 2. 启用校验勾子: 请参考 docs/dev/git_commit_guide.md 中的 Git Hook 设置说明。
